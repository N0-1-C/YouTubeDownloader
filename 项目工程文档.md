## 1. 项目概述

### 1.1 功能定位

一个基于 Web 的 YouTube 视频下载工具，用户通过浏览器界面操作，后端负责视频解析和下载。

### 1.2 核心功能

| 功能 | 描述 |
|------|------|
| 视频解析 | 粘贴 YouTube URL，自动提取标题、缩略图、时长、可用格式 |
| 格式选择 | 展示所有可用画质/编码，支持预设（最佳画质、最佳音频）和手动选择 |
| 实时进度 | 通过轮询 API 获取下载进度（百分比、速度、剩余时间） |
| 格式转换 | 音频提取为 MP3、视频合并为 MP4（依赖 ffmpeg） |
| Cookies 导入 | 手动上传 `cookies.txt` 文件，用于下载会员/受限视频 |
| 代理支持 | 自动检测系统代理，支持 HTTPS/HTTP 代理 |
| 设置管理 | 可配置下载目录、自动打开文件夹 |
| 服务控制 | 网页端一键关闭服务（释放端口） |

### 1.3 系统要求

- **开发运行**: Python 3.8+、Node.js（解决 YouTube n parameter challenge）
- **打包分发**: Windows 10/11 64 位，无需安装任何依赖（内置 ffmpeg + node）
- **网络**: 需要代理/VPN 访问 YouTube

```
│  │ 路由层    │→│ 业务逻辑  │→│ 下载引擎 (yt-dlp) │  │
│  │ /api/*   │  │ 线程管理  │  │ + ffmpeg          │  │
```

### 2.2 技术栈

| 层级 | 技术 | 用途 |
|------|------|------|
| 后端框架 | Flask + Flask-CORS | Web 服务、跨域支持 |
| 下载引擎 | yt-dlp | YouTube 视频解析和下载 |
| 视频处理 | ffmpeg / ffprobe | 格式转换、音视频合并 |
| JS 运行时 | Node.js | 解决 YouTube n parameter challenge |
| 打包工具 | PyInstaller (--onedir) | 打包为独立可分发的 exe |
| 前端 | 原生 HTML/CSS/JS（无框架） | 用户界面 |

### 2.3 数据流

```
用户粘贴 URL → 点击"解析"
    → POST /api/info
        → yt-dlp extract_info(download=False)
        → 返回: video_info + formats[]

用户选择格式 → 点击"开始下载"
    → POST /api/download
        → 创建 task（存入内存 dict）
        → 启动后台线程
            → yt-dlp extract_info(download=True)
            → progress_hooks 回调更新 task 状态
        → 返回: task_id

前端轮询进度
    → GET /api/progress/<task_id>
        → 返回: status, progress, speed, eta, message

下载完成
    → task.status = 'completed'
    → task.filename = 文件路径
    → 自动打开文件夹 (可选)
    → GET /api/file/<task_id> 下载文件

暂停/恢复/取消
    → POST /api/task/<id>/pause  → task.paused = True → DownloadCancelled
    → POST /api/task/<id>/resume → 重新启动下载线程（yt-dlp 自动续传）
    → POST /api/task/<id>/cancel → task.cancelled = True → DownloadCancelled
```

### 4.3 工具路径查找策略

程序按以下优先级查找外部工具：

1. **系统 PATH**：`shutil.which(name)`
2. **应用相对路径**：
   - `tools/ffmpeg/bin/`（打包模式）
   - `ffmpeg/bin/`（备用）
   - `tools/nodejs/`（打包模式）
   - `tools/node/`（备用）
3. **Windows 常见安装路径**：
   - `C:\ffmpeg\bin`
   - `C:\Program Files\nodejs`
   - `C:\Program Files (x86)\nodejs`

找到后自动加入 `os.environ['PATH']`，确保子进程也能使用。

### 4.4 代理检测

代理按以下顺序检测：

1. 环境变量：`HTTPS_PROXY`、`HTTP_PROXY`、`https_proxy`、`http_proxy`
2. Windows 系统代理：`urllib.request.getproxies()`
3. 常见本地端口扫描：`7890`、`11649`、`10808`、`10809`、`1080`、`8080`

#### 5.1.2 URL 验证正则

```python
patterns = [
    r'(https?://)?(www\.)?youtube\.com/watch\?v=[\w-]+',    # 标准视频
```

#### 5.1.3 格式解析逻辑

`get_format_options(url)` 的工作流程：

1. 使用 yt-dlp `extract_info(download=False)` 获取视频信息
2. 构建预设格式：
   - `best`：最佳画质（视频+音频），格式选择器 `bestvideo[ext=mp4]+bestaudio[ext=m4a]/bestvideo+bestaudio/best[ext=mp4]/best`
   - `bestaudio`：最佳音频，下载后通过 `FFmpegExtractAudio` 转为 MP3
3. 遍历 `info['formats']`，过滤纯音频格式（已有预设），去重 `format_id`
4. 简化编码显示：`avc/h264 → H.264`、`hev/h265 → H.265`、`vp9 → VP9`、`av01 → AV1`、`opus → Opus`、`aac → AAC`

#### 5.1.4 下载逻辑

`download_video(task_id, url, format_id, is_audio, save_dir)` 的工作流程：

1. 构建 yt-dlp 配置：
   - `progress_hooks`: 进度回调
   - `outtmpl`: `%(title)s.%(ext)s`
   - `retries: 20`, `fragment_retries: 20`: 增强网络稳定性
   - `http_chunk_size: 10485760` (10MB): 分块下载
   - `throttledratelimit: 100000`: 限速避免反爬
2. 根据 `format_id` 决定下载策略：
   - `best`：合并最佳视频 + 最佳音频 → MP4
   - `bestaudio`：下载最佳音频 → ffmpeg 提取为 MP3
   - 其他 format_id：指定格式 + 合并音频
3. 下载完成后：
   - 查找实际生成的文件路径（音频提取后文件名可能变化）
   - 更新 task 状态为 `completed`
   - 若 `AUTO_OPEN_FOLDER` 为 True，通过 `explorer` 命令打开文件夹

#### 5.1.5 进度回调

`DownloadProgressHook` 类：

- 限制更新频率：每 300ms 最多更新一次
- 状态映射：
  - `downloading`: 计算百分比、速度、ETA
  - `finished`: 100%，提示"正在处理文件"
  - `processing`: 提示"正在处理视频"
  - `error`: 记录错误信息

#### 5.1.6 服务关闭

```python
def shutdown_server():
    import os
    os._exit(0)

@app.route('/api/shutdown', methods=['POST'])
def shutdown():
    threading.Thread(target=shutdown_server, daemon=True).start()
    return jsonify({'success': True, 'message': '服务正在关闭...'})
```

使用 `os._exit(0)` 强制退出，确保端口 5000 被释放。通过 `threading.Thread` 异步执行，确保 HTTP 响应能正常返回。

**任务状态值**:

|--------|------|
| `pending` | 任务已创建，等待开始 |

#### GET `/api/file/<task_id>` — 下载文件

成功时直接返回文件二进制流（`Content-Disposition: attachment`）。

#### GET/POST `/api/save-dir` — 下载目录

**GET 响应**:
```json
{"success": true, "dir": "C:\\Users\\User\\Desktop"}
```

**POST 请求**:
```json
{"dir": "D:\\Downloads"}
```

#### GET/POST `/api/cookies` — Cookies 设置

**GET 响应**:
```json
{"success": true, "cookies_file": "C:\\...\\cookies.txt"}
```

**POST 请求**（上传文件）:
- Content-Type: `multipart/form-data`
- 字段: `cookies_file`（文件）
- 文件保存到应用目录下 `cookies.txt`

**POST 请求**（设置路径，兼容）:
```json
{"cookies_file": "C:\\path\\to\\cookies.txt"}
```

**POST 请求**（清除）:
```json
{"cookies_file": ""}
```

#### POST `/api/shutdown` — 关闭服务

**响应** (200):
```json
{"success": true, "message": "服务正在关闭..."}
```

> 注意：响应返回后服务立即终止，客户端可能收到连接重置错误，这是正常行为。

#### POST `/api/task/<task_id>/pause` — 暂停下载

暂停正在下载的任务。暂停后可通过 resume 接口恢复（yt-dlp 会自动跳过已下载片段）。

**响应** (200):
```json
{"success": true, "message": "正在暂停..."}
```

**错误**: 400（状态不允许暂停）、404（任务不存在）

#### POST `/api/task/<task_id>/resume` — 恢复下载

恢复已暂停的下载任务。重新启动下载线程，yt-dlp 从已下载进度继续。

**响应** (200):
```json
{"success": true, "message": "正在恢复下载..."}
```

**错误**: 400（未暂停状态）、404（任务不存在）

#### POST `/api/task/<task_id>/cancel` — 取消下载

取消下载任务。已暂停的任务立即取消；正在下载的任务设置取消标志等待终止。

**响应** (200):
```json
{"success": true, "message": "正在取消..."}
```

**错误**: 400（已完成等不可取消状态）、404（任务不存在）

1. **解析视频**：点击解析 → 显示 loading → 调用 `/api/info` → 渲染视频信息和格式列表
2. **选择格式**：点击格式卡片，radio 选中效果，`selectedFormat` 变量更新
3. **开始下载**：调用 `/api/download` → 启动进度轮询（`setInterval 500ms`）→ 更新进度条
4. **进度显示**：
   - 进度条宽度 = `progress%`
   - 速度：`formatFileSize(speed) + /s`
   - ETA：`formatETA(eta)` → `mm:ss` 或 `hh:mm:ss`
5. **暂停/恢复/取消**：
   - 暂停：调用 `/api/task/<id>/pause` → 进度条变橙色 → 按钮切换为「▶ 继续」
   - 恢复：调用 `/api/task/<id>/resume` → 进度条恢复红色 → 按钮切换为「⏸ 暂停」
   - 取消：`confirm()` 确认 → 调用 `/api/task/<id>/cancel` → 隐藏进度区域
6. **下载完成**：停止轮询 → 显示文件名 + "下载文件" 按钮 → 可选自动打开文件夹
7. **关闭服务**：`confirm()` 确认 → POST `/api/shutdown` → 页面替换为"服务已关闭"提示

### 7.5 格式列表 UI 优化

- `.format-grid` 设置 `max-height: 320px; overflow-y: auto;`，格式过多时出现滚动条
- `.download-actions` 设置 `position: sticky; bottom: -24px;`，下载按钮始终固定在卡片底部可见
- 滚动条样式自定义（6px 宽、圆角、暗色主题色）

## 10. 关键设计决策

### 10.1 为什么使用 `--onedir` 而非 `--onefile`

- `--onefile` 每次启动需要解压临时文件，启动慢（特别是 300MB+ 的体积）
- `--onedir` 直接从目录加载，启动快
- 目录模式更稳定，不容易出现临时目录权限问题
- 缺点是分发时需要整个文件夹，但通过 ZIP 打包解决

### 10.2 为什么需要 Node.js

YouTube 使用 n parameter (signature function) 来保护视频流。yt-dlp 需要执行 JavaScript 来解析签名。`yt_dlp_ejs` 插件提供了 Node.js 运行时支持，比 PhantomJS 更可靠。

配置方式：
```python
ydl_opts['js_runtimes'] = {'node': {'path': NODE_PATH}}
```

### 10.3 为什么使用 `os._exit(0)` 关闭服务

`sys.exit()` 会触发 Python 的清理流程（atexit handlers、finally blocks），可能导致 Flask 线程无法正常终止。`os._exit(0)` 直接终止进程，确保端口立即释放。

### 10.4 为什么格式列表限制高度

YouTube 视频通常有 20-40 个可用格式，全部展示会导致：
- 页面过长，下载按钮不可见
- 移动端体验差
- 列表滚动不流畅

解决方案：`max-height: 320px` + 滚动条 + sticky 下载按钮。

### 10.5 任务存储为什么用内存字典

- 简单直接，无需数据库
- 任务生命周期短（通常几分钟内完成）
- 服务重启后任务自然清空（符合预期）
- 如果需要持久化，可替换为 SQLite

### 10.6 暂停/恢复的实现原理

yt-dlp 不原生支持暂停/恢复下载。本项目的实现方式：

- **暂停**：在 `DownloadProgressHook` 中检查 `task['paused']` 标志，触发 `DownloadCancelled` 异常中断 yt-dlp。任务状态标记为 `paused`，保留 URL、格式、进度信息。
- **恢复**：用相同的 URL 和参数重新启动下载线程，设置 `nooverwrites=False`，yt-dlp 会自动检测已下载的临时文件并跳过已完成的片段。
- **取消**：类似暂停，但清除进度信息并标记为 `cancelled`。

## 11. 常见问题与排查

### 11.1 `n parameter challenge` 相关错误

**症状**：下载时报 `ERROR: [youtube] n parameter` 错误

**原因**：Node.js 未找到或版本过低

**解决**：
1. 安装 Node.js 18+ 并确保在 PATH 中
2. 或放置 `node.exe` 到 `tools/nodejs/` 目录

### 11.2 视频合并失败

**症状**：下载完成但合并时报错

**原因**：ffmpeg 未找到

**解决**：
1. 安装 ffmpeg 并确保在 PATH 中
2. 或放置 `ffmpeg.exe` + `ffprobe.exe` 到 `tools/ffmpeg/bin/` 目录

### 11.3 代理连接失败

**症状**：解析视频超时或报网络错误

**排查步骤**：
1. 确认代理软件正在运行
2. 检查代理端口是否在检测列表中（7890、11649、10808 等）
3. 或设置环境变量：`set HTTPS_PROXY=http://127.0.0.1:7890`

### 11.4 Cookies 无效

**症状**：导入 cookies 后仍无法下载会员视频

**原因**：
- cookies 已过期
- 导出的 cookies 格式不正确（必须为 Netscape cookies.txt 格式）
- cookies 中缺少关键认证字段

**解决**：重新在 YouTube 登录后导出 cookies

### 11.5 打包后体积过大

**当前体积**：约 322 MB（目录）/ 126 MB（ZIP）

**主要占用**：
- ffmpeg + ffprobe：约 100 MB
- Node.js：约 60 MB
- Python 运行时 + 依赖：约 160 MB

**优化方向**：
- 使用 UPX 压缩 exe（但会增加启动时间）
- 排除更多不必要的 Python 模块
- 精简 Node.js（只保留必需文件）

### 11.6 端口被占用

**症状**：启动时报 `Address already in use`

**解决**：
```bash
# 查找占用端口的进程
netstat -ano | findstr :5000
# 终止进程
taskkill /F /PID <PID号>
```

或通过网页端的"关闭服务"按钮。
